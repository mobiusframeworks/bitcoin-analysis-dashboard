üîÑ Halving Cycle Features: Sine/Cosine Transformations

Bitcoin's ~4-year halving cycle creates predictable patterns in price behavior. We engineered features to capture this cyclical nature using sine and cosine transformations of the halving cycle position.

Why Sine/Cosine Transformations?

Traditional linear features like "days since halving" fail to capture the cyclical nature of Bitcoin's 4-year pattern. Sine and cosine transformations convert the linear timeline into circular coordinates, allowing the model to recognize that day 0 (halving) and day 1460 (next halving) are adjacent points in the cycle, not far apart.

Mathematical Formulation:
‚Ä¢ Cycle Position = days_since_halving / 1460 (normalized 0 to 1)
‚Ä¢ halving_sin = sin(2œÄ √ó Cycle Position)
‚Ä¢ halving_cos = cos(2œÄ √ó Cycle Position)
‚Ä¢ halving_sin_quarterly = sin(8œÄ √ó Cycle Position) [captures sub-cycles]
‚Ä¢ halving_cos_quarterly = cos(8œÄ √ó Cycle Position)

Together, sine and cosine provide a complete 2D representation of the cycle position, capturing both the phase (where we are in the cycle) and the rate of change (how fast we're moving through phases).
